# API Contract Specification: Overlayer

## Overview

This defines the .NET Minimal API surface. The API is strictly stateless. All endpoints require an `X-Session-ID` header, representing an anonymous user session. There is no relational database; S3 is the singular source of truth.

## Base Configuration

- **Content-Type:** `application/json`
- **Authentication:** None.
- **Authorization:** Session Isolation via `X-Session-ID` HTTP Header (UUID v4 format).
- **Job Identification:** `jobId` is a client-generated UUID v4, validated server-side.

---

## 1. Request Upload URLs

Generates strict AWS S3 Presigned POST data for the client to upload their base video and overlay image directly to the bucket. The Presigned POST policies include a `content-length-range` condition derived from server-side configurable upload limits, preventing oversized files from being accepted at the AWS edge.

**Endpoint:** `GET /api/jobs/{jobId}/upload-urls`
_(Note: `jobId` is a UUID v4 generated by the client to allow for optimistic UI updates)._

**Headers:**

- `X-Session-ID`: `string` (Required) - e.g., `123e4567-e89b-12d3-a456-426614174000`

**Request Body:** `Empty`

**Response: 200 OK**

```json
{
  "jobId": "abc-123",
  "videoUpload": {
    "url": "https://my-bucket.s3.amazonaws.com/",
    "fields": {
      "key": "jobs/session-id/job-id/overlay.png",
      "xAmzCredential": "...",
      "xAmzAlgorithm": "...",
      "xAmzDate": "...",
      "policy": "...",
      "xAmzSignature": "...",
      "xAmzSecurityToken": "...",
      "contentType": "video/"
    },
    "maxFileSize": 10485760
  },
  "overlayUpload": {
    "url": "https://my-bucket.s3.amazonaws.com/",
    "fields": {
      "key": "jobs/session-id/job-id/overlay.png",
      "xAmzCredential": "...",
      "xAmzAlgorithm": "...",
      "xAmzDate": "...",
      "policy": "...",
      "xAmzSignature": "...",
      "xAmzSecurityToken": "...",
      "contentType": "image/png"
    },
    "maxFileSize": 4194304
  }
}
```

_(Note: `maxFileSize` is in bytes. These values are derived from the server-side `UploadLimitsOptions` configuration and are included so the frontend can validate file sizes client-side before upload. The same limits are enforced at the S3 edge via the Presigned POST `content-length-range` policy condition.)_

**Response: 400 Bad Request**

- Missing or invalid `X-Session-ID` header.
- Invalid `jobId` format (must be UUID v4).

## 2. List All Jobs

Fetches all jobs for the user's session, encompassing both pending and completed items, by evaluating the inferred state from S3 prefixes.

**Endpoint:** `GET /api/jobs`

**Headers:**

- `X-Session-ID`: `string` (Required)

**Response: 200 OK**

```json
{
  "jobs": [
    {
      "jobId": "abc-123",
      "status": "PROCESSING",
      "createdAt": "2026-04-14T10:00:00Z",
      "downloadUrl": null
    },
    {
      "jobId": "xyz-987",
      "status": "COMPLETED",
      "createdAt": "2026-04-14T09:15:00Z",
      "downloadUrl": "https://my-bucket.s3.amazonaws.com/processed/123e4567.../xyz-987/output.mp4?X-Amz-Signature=..."
    },
    {
      "jobId": "err-456",
      "status": "FAILED",
      "createdAt": "2026-04-14T08:00:00Z",
      "downloadUrl": null,
      "reason": "Video format not supported"
    },
    {
      "jobId": "inc-789",
      "status": "MISSING_ASSETS",
      "createdAt": "2026-04-14T10:05:00Z",
      "downloadUrl": null
    }
  ]
}
```

_(Note: `downloadUrl` is an S3 Presigned GET URL with a short expiration and is present only when status is `COMPLETED`. `createdAt` is derived from the S3 `LastModified` timestamp of the earliest uploaded input file for the job.)_

**Response: 400 Bad Request**

- Missing or invalid `X-Session-ID` header.

---

## Standard Error Response Format

All endpoints return errors using the HTTP Problem Details standard shape, natively supported by ASP.NET Minimal APIs:

```json
{
  "type": "https://tools.ietf.org/html/rfc9110#section-15.5.1",
  "title": "Bad Request",
  "status": 400,
  "detail": "The X-Session-ID header is missing or not a valid UUID v4."
}
```
